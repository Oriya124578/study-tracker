# Handoff Report: Milestone 4 Explorer

## 1. Observation
- **Scope Contract:** `SCOPE.md` specifies extending `cl_personalTasks` with `recurrence: { type, interval, until, ... }` and handling `cl_aiSuggestions`.
- **AI Manager Backend:** In `functions/index.js` (lines 28-52), `exports.aiManager` is scheduled via `onSchedule("0 7,21 * * *")` but lacks a timezone parameter (like `timeZone: "Asia/Jerusalem"`) and currently writes a hardcoded string `"Review your schedule for the day."` without calling any AI for contextual generation.
- **AI Manager Frontend:** `src/lib/firestoreRepo.js` (lines 347-355) implements `subscribeAiSuggestions`, but it is completely unused in `src/store/useStore.js` and the Manager screen (`src/components/command-center/CommandCenterView.jsx`).
- **Recurring Tasks:** `src/components/tasks/TasksView.jsx` (lines 331-650) and `src/lib/recurrence.js` implement recurrence using a completely separate collection `cl_recurringTasks`. This contradicts the `ORIGINAL_REQUEST.md` and `SCOPE.md` which require adding a `recurrence` field directly to existing tasks (`cl_personalTasks`) and generating future instances with override capabilities.

## 2. Logic Chain
- To fulfill the global 07:00 & 21:00 requirement, the `aiManager` Cloud Function in `functions/index.js` must be updated with `timeZone: "Asia/Jerusalem"`. It must also be wired to a real AI prompt to generate the suggestion content based on user context.
- For the AI suggestions UI to work, `src/store/useStore.js` must be updated to hold a `pendingSuggestions` state populated via `subscribeAiSuggestions`. The `CommandCenterView.jsx` component must then render these suggestions and allow "Accept" or "Reject" mutations (via `updateAiSuggestion`).
- For recurring tasks, the current separate `cl_recurringTasks` implementation needs to be refactored. The UI in `TasksView.jsx` should be updated so users can define recurrence inline on a `cl_personalTasks` item. The `recurrence.js` logic needs to be adapted to auto-instantiate future task documents in `cl_personalTasks` when the user views the task list or calendar, and support "override" logic ("רק זו" / "כל ההבאות").

## 3. Caveats
- Generating task instances on the fly vs. pre-generating them in the DB: If instances are pre-generated by a Cloud Function or on the client side, we must manage their synchronization. Modifying "only this" requires creating a standalone document, while modifying "all following" requires updating the base rule and rewriting future generated documents. The exact implementation strategy for "רק זו" vs "כל ההבאות" was not detailed in the codebase yet.
- Connecting the Cloud Function to Gemini might require passing API keys or using Firebase Extensions (Vertex AI).

## 4. Conclusion
1. **Recurring tasks logic**: Implement inside `cl_personalTasks` by adding a `recurrence` object. Refactor `TasksView.jsx` and `src/lib/recurrence.js` to remove `cl_recurringTasks` and instead manage instances of `cl_personalTasks`. Implement instance-override logic.
2. **AI Manager Cloud Function**: Edit `functions/index.js` to add `timeZone: "Asia/Jerusalem"` and implement logic to contextually generate recommendations via an LLM.
3. **AI suggestions UI**: Hook up `cl_aiSuggestions` in `src/store/useStore.js` and render them in `src/components/command-center/CommandCenterView.jsx` with Accept/Reject controls.

## 5. Verification Method
- **Recurring Tasks:** Verify by editing a personal task, setting a recurrence (e.g., daily), and checking if `cl_personalTasks` contains the `recurrence` field. Edit a future occurrence to test the "only this" override.
- **AI Manager CRON:** Deploy functions and verify in the Firebase Console that the cron job schedule explicitly lists the `Asia/Jerusalem` timezone.
- **AI Suggestions UI:** Manually insert a document into `cl_aiSuggestions` with `status: 'pending'`. Open the Command Center (Manager UI) and verify the suggestion appears and can be accepted/rejected, updating the Firestore document accordingly.
